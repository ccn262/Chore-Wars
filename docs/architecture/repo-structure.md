# Repo Structure

## Current repository intent

This repository starts as a documentation-first planning space and later grows into the app implementation.

## Directory purposes

- `docs/product/`: product vision, scope, and roadmap
- `docs/architecture/`: technical structure and boundaries
- `docs/ui/`: design system and interaction guidance
- `docs/database/`: schema and data governance
- `docs/engineering/`: build, test, deploy, and security rules
- `docs/market/`: market positioning and launch plans
- `docs/prompts/`: reusable Codex prompts and task templates
- `docs/logs/`: decision records and session handoff notes
- `docs/operations/`: service setup and release checklists
- `supabase/`: database migrations, seed data, functions, and policies
- `emails/`: email templates and related assets later
- `public/`: static assets later
- `tests/`: test fixtures and future test assets

## Growth rule

Add new top-level folders only when they represent a durable system boundary.
