# Routing Map

## Route principles

- The home screen should be the primary landing surface
- Routes should be short, obvious, and touch-friendly
- Avoid burying the main chore action behind deep navigation
- Keep reporting separate from action, but easy to reach

## Early route set

- `/` Home and hero screen
- `/household/[id]` Household overview
- `/chore/[id]` Chore detail or quick action surface
- `/reporting` Visual progress and history
- `/settings` Household and preference settings
- `/onboarding` First-run setup

## Route design notes

- Prefer a few clear routes over many nested pages
- Use modal or bottom-sheet style interactions when they shorten the path
- On mobile, favour in-place actions over page-hopping
