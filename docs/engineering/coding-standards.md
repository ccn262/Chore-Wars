# Coding Standards

## General standards

- Prefer clarity over cleverness
- Keep functions small and focused
- Name things for the domain, not for the implementation detail
- Avoid hardcoded user-facing strings when a translation path exists

## UI standards

- Use mobile-first layouts
- Keep the main action obvious
- Avoid dense visual noise

## Data standards

- Handle dates and times with locale awareness
- Avoid currency assumptions
- Prefer explicit types and clear data contracts
