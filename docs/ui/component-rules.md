# Component Rules

## Recommended component patterns

- Large action cards
- Reward chips
- Progress rings or bars
- Bottom sheets for quick actions
- Compact member avatars or badges

## Do

- Reuse components consistently
- Keep components small and composable
- Make touch targets generous
- Keep text content easy to translate

## Do not

- Build dense table-heavy components for the main experience
- Hide essential actions in menus
- Overload cards with too much metadata
- Hardcode copy into deeply nested components unless necessary
