# Interaction Rules

## Core interaction rule

The user should be able to open the app, tap a chore, and get points within 3 seconds.

## Behaviour rules

- Keep the main completion flow to as few taps as possible
- Confirm actions with lightweight feedback, not blocking dialogs
- Use bottom sheets or inline surfaces for fast decisions
- Make destructive actions rare and clearly labelled
- Prefer one-tap chore completion
- Use quick press button selections for chores
- Keep optional proof or photo capture secondary to the main action
- Avoid clutter and unnecessary friction
- Avoid desktop-style dashboards and admin-heavy flows
- Prioritise fast scanning and instant feedback

## Mobile ergonomics

- Place frequent actions within thumb reach
- Avoid tiny hit areas
- Avoid interactions that require precise pointer movement
- Use large touch targets and big rounded buttons
- Keep navigation app-like and suitable for bottom navigation

## Reporting interactions

- Reports should use cards, charts, badges, streaks, and visual summaries
- Reporting should be easy to scan on a phone
- Reporting should not feel like a spreadsheet or admin console
